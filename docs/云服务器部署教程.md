# Aura 云服务器部署完整教程

从当前本地开发状态到云端正式运行的一步步指南。

---

## 一、部署架构概览

```
┌─────────────────┐      HTTPS       ┌──────────────────────┐
│  微信小程序      │ ──────────────►  │  云服务器              │
│  (前端)          │                  │  ├─ Nginx (反向代理)   │
└─────────────────┘                  │  ├─ aura-backend      │
                                      │  └─ MySQL (可选)       │
                                      └──────────────────────┘
```

**需要部署的：**
- `aura-backend`（Node.js 后端）→ 云服务器
- 域名 + HTTPS 证书
- 微信小程序 request 合法域名配置

**不需要部署的：**
- `aura-miniapp` 编译后上传到微信，由微信 CDN 分发，无需自建前端服务器

## 三、云服务器选择与基础配置

### 3.1 购买云服务器

**推荐配置（入门）：**
- CPU：1 核
- 内存：2GB
- 系统盘：40GB
- 系统：Ubuntu 22.04 LTS（或 CentOS 7+）

**云厂商示例：**
- 阿里云 ECS
- 腾讯云 CVM
- 华为云弹性云服务器

### 3.2 安全组开放端口

在云控制台「安全组」中放行：

- **22**：SSH 登录
- **80**：HTTP（用于申请证书或重定向）
- **443**：HTTPS

### 3.3 用 SSH 连接服务器

```bash
ssh root@你的服务器公网IP
# 输入密码或使用密钥
```

## 四、服务器环境安装

### 4.1 安装 Node.js（推荐 v18 或 v20）

```bash
# 使用 NodeSource 安装（Ubuntu）
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs

# 验证
node -v   # 应显示 v20.x.x
npm -v
```

### 4.2 安装 Nginx（反向代理 + HTTPS）

```bash
sudo apt update
sudo apt install -y nginx
sudo systemctl enable nginx
```

### 4.3 安装 PM2（进程守护）

```bash
sudo npm install -g pm2
```

### 4.4（可选）安装 MySQL

若需持久化用户数据：

```bash
sudo apt install -y mysql-server
sudo mysql_secure_installation  # 按提示设置 root 密码
```

## 五、后端代码部署

### 5.1 上传代码到服务器

**方式一：Git（推荐）**

本地项目推送到 GitHub/Gitee，在服务器：

```bash
cd /var/www
sudo mkdir -p aura
sudo chown $USER:$USER aura
cd aura
git clone https://github.com/你的用户名/aura.git .
# 或：仅 clone aura-backend 目录
```

**方式二：SCP 上传**

```bash
# 在本地项目根目录执行
scp -r aura-backend root@你的服务器IP:/var/www/aura/
```

### 5.2 配置环境变量

```bash
cd /var/www/aura/aura-backend
nano .env   # 或 vi / vim
```

填入（根据实际修改）：

```env
MOONSHOT_API_KEY=你的Kimi真实Key
MOONSHOT_BASE_URL=https://api.moonshot.cn/v1
MOONSHOT_MODEL=moonshot-v1-32k

PORT=8787
HOST=0.0.0.0

# MySQL（若已安装）
MYSQL_HOST=localhost
MYSQL_PORT=3306
MYSQL_USER=root
MYSQL_PASSWORD=你的MySQL密码
MYSQL_DATABASE=aura_app
```

保存后，确保 `.env` 在 `aura-backend` 目录下，且不会被提交到 Git（已在 `.gitignore`）。

### 5.3 安装依赖并试运行

```bash
cd /var/www/aura/aura-backend
npm install --production

# 若使用 MySQL，初始化数据库
npm run db:init

# 直接测试
npm run start
```

另开一个终端，测试：

```bash
curl http://127.0.0.1:8787/health
# 应返回 {"ok":true,"service":"aura-backend",...}
```

按 `Ctrl+C` 停止，接下来用 PM2 常驻运行。

### 5.4 用 PM2 启动

```bash
cd /var/www/aura/aura-backend
pm2 start src/server.js --name aura-backend
pm2 save
pm2 startup   # 按提示执行输出命令，实现开机自启
```

## 六、域名与 HTTPS

### 6.1 域名解析

在域名服务商处添加 A 记录：

- 主机记录：`api`（或 `aura`，随你喜欢）
- 记录值：服务器公网 IP

例如：`api.yourdomain.com` → `123.45.67.89`

### 6.2 申请 SSL 证书

**免费（Let's Encrypt）：**

```bash
sudo apt install -y certbot python3-certbot-nginx
sudo certbot --nginx -d api.yourdomain.com
# 按提示输入邮箱、同意条款，自动配置 Nginx
```

**云厂商证书：** 在控制台申请、下载 Nginx 格式证书，上传到服务器。

### 6.3 配置 Nginx 反向代理

```bash
sudo nano /etc/nginx/sites-available/aura-api
```

内容示例：

```nginx
server {
    listen 80;
    server_name api.yourdomain.com;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name api.yourdomain.com;

    ssl_certificate /etc/letsencrypt/live/api.yourdomain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/api.yourdomain.com/privkey.pem;

    location / {
        proxy_pass http://127.0.0.1:8787;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

启用配置：

```bash
sudo ln -s /etc/nginx/sites-available/aura-api /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

### 6.4 验证

浏览器访问 `https://api.yourdomain.com/health`，应返回 JSON 健康检查结果。

## 七、微信小程序配置

### 7.1 配置 request 合法域名

1. 登录 [微信公众平台](https://mp.weixin.qq.com)
2. 开发 → 开发管理 → 开发设置 → 服务器域名
3. 在 **request 合法域名** 中添加：`https://api.yourdomain.com`
4. 保存（需管理员扫码确认）

### 7.2 修改前端 API 地址

编辑 `aura-miniapp/src/utils/api.ts`：

```typescript
const API_BASES: Record<AuraApiMode, string> = {
  local: "http://127.0.0.1:8787",
  remote: "https://api.yourdomain.com",  // ← 改成你的域名
};
```

### 7.3 默认使用 remote

在 `api.ts` 中修改 `DEFAULT_MODE`，或发布前在代码里写死使用 `remote`：

```typescript
const DEFAULT_MODE: AuraApiMode = "remote";  // 正式环境用 remote
```

或保持 `local`，通过「设置页/调试入口」让用户手动切换为 `remote` 并保存到本地存储。

### 7.4 构建并上传小程序

```bash
cd aura-miniapp
npm run build:mp-weixin
```

在微信开发者工具中：

1. 打开 `dist/build/mp-weixin` 目录
2. 右上角「上传」→ 填写版本号、备注
3. 登录公众平台 → 版本管理 → 提交审核 → 发布

## 九、常见问题

### Q1：请求一直失败，提示「连接中断」

- 确认服务器域名已在微信公众平台配置
- 确认使用 HTTPS，且证书有效
- 开发阶段可在开发者工具勾选「不校验合法域名」

### Q2：本地可以，真机不行

- 真机必须走 HTTPS + 合法域名
- 检查 `api.ts` 中 `remote` 是否写对、是否为 HTTPS

### Q3：PM2 进程重启后消失

执行并完成 `pm2 startup` 提示的 `sudo env PATH=...` 命令，才能开机自启。

### Q4：502 Bad Gateway

- 后端未启动：`pm2 restart aura-backend`
- 端口不符：确认 Nginx `proxy_pass` 指向的端口与 `PORT` 一致

### Q5：MySQL 连接失败

- 检查 `.env` 中 `MYSQL_*` 配置
- 云服务器需开放 3306（仅本机访问时可不开外网）
- 未配置 MySQL 时，数据接口会 503，前端会退回本地存储
